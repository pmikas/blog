Mushroom Classification


This dataset was originally contributed to the UCI Machine Learning repository nearly 30 years ago, and was obtained from Kaggle database.
It includes descriptions of hypothetical samples corresponding to 23 species of gilled mushrooms in the Agaricus and Lepiota Family Mushroom drawn from The Audubon Society Field Guide to North American Mushrooms (1981). Each species is identified as definitely edible, definitely poisonous, or of unknown edibility and not recommended. This latter class was combined with the poisonous one.

Objective


Our objective is to learn how to deal with categorical features, and study classification models.

Content


This kernel is divided into:

  1. Basic information

3. Model Training¶

The model is created through the following steps:

  • Split the dataset
  • Build model pipelines
  • Declare hyperparameters o tune
  • Fit and tune models with cross-validation
  • Evaluate metrics and select winner

Split the dataset


The target variable (y) and the input features (X) should be separated into separate objects.

In [8]:
# Create separate object for target variable
y = df.class_p

# Create separate object for input features
X = df.drop('class_p', axis = 1)

Training sets are used to fit and tune the models and test sets are put aside as unseen data to evaluate your models. Thus, as we have only one dataset, the train and test sets should be separate into separate variables.

  • test_size = 0.2: set aside 20% of the observations for the test set
  • random_state = 1234: set for replicable results
  • stratify = df.class_p: make sure the target variable's classes are balanced in each subset of data

Declare hyperparameters to tune


Hyperparameters express "higher-level" structural settings for modeling algorithms.

  • C is the strength of the penalty, inverse of alpha from regularization strength
  • Higher values of C mean weacker penalties
  • C is a positive value, typically between 0 and 1000
  • The default is 1.0

Fit and tune models with cross-validation


The GridSearchCV function performs cross-validation on the hyperparameter grid, through each combination of values. It then calculates cross-validated scores (using performance metrics) for each combination of hyperparameter values and picks the combination that has the best score.

  • cv is the number of cross-validation folds
  • n_jobs = -1 trains in parallel across the maximum number of cores of the computer, speeding it up
In [15]:
# Create empty dictionary called fitted_models
fitted_models = {}

# Loop through model pipelines, tuning each one and saving it to fitted_models
for name, pipeline in pipelines.items():
    
    # Create cross-validation object from pipeline and hyperparameters
    model = GridSearchCV(pipeline, hyperparameters[name], cv = 10, n_jobs = -1)
    
    # Fit model on X_train, y_train
    model.fit(X_train, y_train)
    
    # Store model in fitted_models[name]
    fitted_models[name] = model
    
    # Print when the model is fitted
    print(name, 'has been fitted.')

Evaluate models and select winner


First evaluate the models by looking at their cross-validated performance on the training set, through the holdout accuracy scores

  • Accuracy is simply the percent of observations correctly classified by the model
  • Because is the average accuracy from the holdout folds, higher is almost always better
In [16]:
# Display best_score_ for each fitted_model
for name, model in fitted_models.items():
    print(name, model.best_score_)

Straight accuracy is not always the best way to evaluate a classification model, specifically when evaluating imbalanced classes in the target variable.

Area under ROC curve is the most reliable metric for classification tasks. It is equivalent to the probability that a randomly chosen positive observation ranks higher (has a higher predicted probability) than a randomly chosen negative observation.

  • ROC curve is a way to visualize the relationship between TPR (true positive rate) and FPR (false positive rate) for classification models
  • Plot the TPR and FPR at different thresholds
In [17]:
# Loop through the fitted_models to predict probabilities
for name, model in fitted_models.items():
    pred = model.predict_proba(X_test)
    
    # Get just the prediction for the positive class (1)
    pred = [p[1] for p in pred]
    
    # Calculate ROC curve from y_test and pred
    fpr, tpr, thresholds = roc_curve(y_test, pred)
    
    # Calculate and print AUROC
    print(name, auc(fpr, tpr))
